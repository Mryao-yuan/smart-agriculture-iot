# 智慧温室 IoT 平台软件说明手册

## 1. 软件概述

本系统是一个面向温室场景的智慧农业管理平台，集成了：

- 温室运行状态总览
- 单棚设备沙盘与设备控制
- 跨棚历史数据对比
- 多维数据分析与导出
- 批次管理与电子工单
- 多设备联控
- 阈值报警与消息推送

系统前端基于 `Streamlit`，后台依赖：

- IoT 平台接口
- TiDB / MySQL 兼容数据库
- Plotly 图表

---

## 2. 启动方式

### 2.1 环境要求

- Python 3.10 及以上
- 已安装 `requirements.txt` 中依赖
- 已正确配置：
  - `streamlit secrets`
  - IoT 平台账号信息
  - 数据库连接信息

### 2.2 安装依赖

```bash
pip install -r requirements.txt
```

### 2.3 启动平台

```bash
streamlit run app.py
```

## 4. 功能说明

## 4.1 设备整体状态

该页面用于查看平台总体运行状态，主要包括以下功能：

### 4.1.1 天气横幅与天气预警参数设置

- 自动读取首个设备所在位置的天气信息
- 展示：
  - 天气
  - 温度
  - 湿度
  - 风力
- 可配置天气预警阈值：
  - 高温
  - 低温
  - 高湿
  - 低湿
  - 大风
- 可启用或关闭天气预警

### 4.1.2 温室地图状态

- 基于设备经纬度展示温室位置
- 按颜色区分状态：
  - 绿色：正常
  - 红色：报警
  - 黄色：离线
  - 灰色：停用
- 鼠标悬浮可查看：
  - 温室名称
  - 在线状态
  - 报警状态
  - 节点数

### 4.1.3 进入单棚设备沙盘

- 可通过以下方式进入单棚设备沙盘：
  - 点击地图上的温室点
  - 点击温室卡片下方按钮
- 进入后会自动切换到 `🎮 单棚设备沙盘` 页面

### 4.1.4 跨棚数据对比

- 可选择某一类环境指标
- 支持按以下范围进行对比：
  - 当前 1 小时
  - 今日
  - 最近一周
  - 最近一月
- 多个温室的历史趋势会绘制在同一张曲线图中

适用于：

- 查看 9 个温室中某一指标的整体趋势
- 发现异常棚区
- 进行横向运行对比

---

## 4.2 单棚设备沙盘

该页面用于查看某一座温室的详细设备分布、实时状态及控制能力。

### 4.2.1 单棚切换

- 可在页面顶部选择目标温室
- 也可从首页地图点击后自动进入该温室

### 4.2.2 温室设备沙盘

- 平面化展示温室设备分布
- 按设备位置逻辑进行区分：
  - 顶部棚膜区
  - 中部环境区
  - 地面根区
  - 侧边辅助区

### 4.2.3 设备详情查看

- 点击沙盘设备节点后，右侧展示设备详情
- 可查看：
  - 传感器名称
  - 传感器 ID
  - 更新时间
  - 当前值

### 4.2.4 单设备控制

对于开关型设备（如卷膜、雾化、补光等），右侧支持控制面板操作。

当前系统将以下类型视为可控设备：

- `sensorTypeId = 2`
- `sensorTypeId = 5`
- `sensorTypeId = 6`

控制逻辑说明：

- 若 `DEBUG_MODE = True`
  - 不真实下发
  - 仅展示模拟下发信息
- 若 `DEBUG_MODE = False`
  - 调用真实 IoT 控制接口下发命令

### 4.2.5 控制动作联动农事记录

部分控制动作会自动触发表单联动，例如：

- 雾化 / 喷淋 / 水泵：
  - 触发“降温 / 加湿”记录
- 卷膜 / 卷被 / 顶窗 / 通风：
  - 触发“通风 / 保温”记录

填写后可直接归档到工单系统。

---

## 4.3 多维数据分析

用于做历史数据分析、区域聚合分析和历史数据导出。

### 4.3.1 传感器参数相关性分析

- 选择某个温室
- 选择两个传感器指标
- 系统绘制散点图与趋势线
- 自动计算相关性

适合分析：

- 光照 vs 空温
- 土壤 EC vs PH
- 湿度 vs 温度

### 4.3.2 前中后区域聚合分析

- 基于命名规则识别：
  - 前区
  - 中区
  - 后区
- 对指定指标做区域均值聚合
- 支持多棚横向对比

### 4.3.3 历史数据导出

- 支持按以下条件提取历史数据：
  - 温室
  - 传感器
  - 时间范围
- 查询后可直接导出 CSV

## 4.4 批次工单与联控

该页面围绕“生产过程管理”展开，包括批次、工单、联控三部分。

### 4.4.1 批次概览

- 展示所有运行中 / 已结束批次
- 支持查看：
  - 作物
  - 品种
  - 当前阶段
  - 已种植天数
  - 工单数量
  - 最近工单
  - 采收倒计时

### 4.4.2 新增批次

支持创建新的种植批次，包括：

- 温室
- 作物名称
- 品种名称
- 初始阶段
- 定植 / 播种日期
- 预计采收日期

### 4.4.3 工单中心

支持：

- 标准化电子工单录入
- 工单筛选
- 工单查询
- 工单导出
- 工单删除

工单时间规则：

- 点击“生成并归档”时刻视为工单完成时间
- 若有耗时，则自动反推出开始时间
- 若为联控动作工单，则视为瞬时动作，耗时显示为 `-`

### 4.4.4 批次环境回溯

- 选择批次后自动截取其周期内的环境历史数据
- 可选择环境指标
- 可按不同时间粒度聚合
- 自动叠加生长阶段标注

### 4.4.5 多设备一键联控

支持：

- 默认全选所有在线温室
- 自动识别某类可控设备
- 一次性下发统一动作
- 查看执行明细
- 将联控动作自动归档为工单

---

## 4.5 策略与预警

用于配置设备 / 传感器的阈值报警策略。

### 4.5.1 通知方式

支持：

- 官方运维群
- 自定义钉钉机器人 Webhook

### 4.5.2 报警策略配置

可按以下两类设备配置规则：

- 数值型传感器
- 开关 / 状态型传感器

支持按：

- 单棚配置
- 全棚批量部署

### 4.5.3 当前报警逻辑

系统已采用更合理的状态机逻辑：

- 边缘触发
  - 仅在正常跨越到异常时报警
- 冷却提醒
  - 持续异常时按冷却周期重复提醒
- 恢复通知
  - 恢复正常后发送恢复消息并重置状态

当前推送方式为：

- 钉钉消息推送

## 6. 控制模式说明

平台使用 `utils/controllers.py` 中的全局开关：

```python
DEBUG_MODE = True
```

说明如下：

- `True`
  - 控制只模拟，不真实下发
  - 用于调试和演示
- `False`
  - 调用真实控制接口
  - 下发到硬件设备

## 7. 典型使用流程

### 场景一：查看全局温室状态

1. 进入 `🌐 设备整体状态`
2. 查看天气与预警横幅
3. 查看地图上各温室运行状态
4. 判断哪些棚处于报警或离线状态

### 场景二：进入单棚设备沙盘

1. 在首页地图选择某一座温室
2. 自动切到 `🎮 单棚设备沙盘`
3. 在沙盘中点选具体设备
4. 查看右侧实时详情或执行控制

### 场景三：执行联控

1. 进入 `📋 批次工单与联控`
2. 选择目标温室
3. 选择联控动作
4. 点击下发
5. 查看执行明细
6. 需要时归档联控工单

### 场景四：配置报警

1. 进入 `⚙️ 策略与预警`
2. 选择通知方式
3. 选择应用范围
4. 选择监控指标
5. 填写阈值
6. 部署策略

## 8. 当前已知限制

当前版本仍有以下边界：

- 摄像头流已预留数据表，但未正式接入页面
- 温室沙盘属于“业务示意布局”，不是真实三维建模
- 地图点击切换与页面切换已打通，但仍依赖 `session_state` 驱动
- 报警当前推送通道以钉钉为主，短信 / 微信未接入

---

## 9. 运维建议

- 正式环境建议将 `DEBUG_MODE` 设为 `False`
- 建议定时运行同步任务：
  - IoT 数据同步
  - 报警巡检
- 建议定期检查：
  - `alert_state` 是否正常更新
  - `sensor_history` 是否持续入库
  - `work_orders` / `batches` 是否存在脏数据
